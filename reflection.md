# Reflection (Prompt Engineering Assignment)

Building this persona-based chatbot helped me realize that prompt engineering is not about writing complicated prompts, but about carefully designing model behavior. In the beginning, my prompts were very short and generic, so the responses also felt generic. Although the answers were technically correct, they did not reflect the intended persona clearly. After adding detailed persona descriptions, communication patterns, and behavioral constraints, the responses became much more accurate and personality-driven.

This strongly demonstrated the GIGO principle: weak prompts produce weak outputs.
One of the most impactful improvements was introducing few-shot examples within the system prompts. These examples gave the model a clearer understanding of tone, structure, and reasoning style. For Anshuman Singh, examples centered around systems thinking and bottleneck analysis produced a more analytical response style. For Abhimanyu Saxena, examples emphasizing rapid execution and iteration created practical, action-focused outputs. For Kshitij Mishra, examples built around calm guidance and fundamentals-first teaching resulted in patient, mentor-like responses. This showed me that examples are among the most effective ways to guide LLM behavior.

Another major takeaway was the importance of constraints. I clearly specified what each persona should avoid, such as unnecessary hype, overly aggressive language, or vague theoretical explanations without actionable advice. These restrictions helped reduce style drift and made the chatbot more consistent across different prompts. Output formatting rules also improved the interaction by keeping responses concise and ending them with a question, which made conversations feel more engaging and structured.

From a product perspective, I learned that strong prompts alone are not sufficient; the user experience also plays a significant role. Features like persona switching with chat reset, suggestion chips, and typing indicators made the application easier to interact with and evaluate. Proper error handling was equally important because API failures should not break the user experience.

If I continue developing this project further, I would implement persistent conversation memory for each persona, add streaming responses to make conversations feel more natural, and include lightweight analytics for comparing response quality across prompts. I would also design a formal prompt evaluation framework with fixed test questions to measure consistency, tone alignment, and actionability. Overall, this project taught me that effective AI behavior comes from clear objectives, structured context, thoughtful constraints, and continuous prompt refinement.
